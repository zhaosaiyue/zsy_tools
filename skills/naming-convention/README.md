# 项目架构与命名规范审查专家

项目架构与命名规范审查专家。审查目录、文件、变量、类名是否符合规范，并为新组件生成 3-5 个候选命名。

## 触发方式

关键词触发：

> 命名 / 重命名 / 怎么命名 / 文件名 / 目录名 / 变量名 / 类名 / 规范审查 / 命名检查

## 命名规范速查

| 对象类型 | 规范 | 示例 |
|----------|------|------|
| 目录 / 项目 | 短横线小写（如 `sign-ai-plugin`） | `sign-ai-plugin` |
| 代码文件（py/js/ts/json/sql） | 短横线小写 | `json-tool.py`, `build-index.js` |
| 文档文件（md） | 全大写下划线 | `SKILL.md`, `README.md` |
| 类 / 接口 | 大驼峰 | `BillDetailService` |
| 方法 / 变量 | 小驼峰 | `getBillDetail()`, `mockData` |
| 常量 | 全大写下划线 | `MAX_RETRY_COUNT` |

## 两种使用模式

1. **审查模式**：提供现有文件/目录结构，输出不合规项及修改建议
2. **命名建议模式**：描述新组件的业务语义，输出 3-5 个符合规范的候选名
