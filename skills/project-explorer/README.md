# project-explorer

新项目全面探索工具。对陌生代码仓库进行两阶段扫描，输出完整 Markdown 概览报告。

## 触发词

> 读项目、认识项目、接手项目、项目概览、project explorer、探索项目、扫描项目、项目分析

## 功能

### Phase 1 — 广度快照
- 目录结构（3层树）
- 技术栈 + 框架识别（自动检测 package.json / pom.xml / go.mod / requirements.txt 等）
- 核心依赖清单
- 接口文件 / 数据模型文件 / 配置文件 / 测试文件数量统计

### Phase 2 — 深度分析
- **API 接口**：逐文件提取 HTTP 方法、路径、入参、功能
- **HTTP 外部调用**：项目主动发起的第三方/微服务调用
- **数据模型**：逐表提取字段名、类型、约束、关联关系
- **业务逻辑摘要**：每个 Service 的职责、外部依赖、关键业务规则
- **配置 & 环境变量**：关键配置项和 .env 变量清单

### 报告结构
1. 技术栈 & 依赖
2. 项目结构 & 模块划分
3. API 接口列表
4. HTTP 外部调用
5. 数据模型 & 数据库
6. 配置 & 环境变量
7. 核心业务逻辑摘要
8. 代码规范 & 工程实践
9. 待核查项
10. 快速上手建议

## 快捷指令

| 指令 | 效果 |
|------|------|
| `全自动` | 自动完成 Phase 1 + Phase 2，无需确认 |
| `只要快照` | 仅 Phase 1，不精读代码，快速生成报告 |
| `深入 {模块名}` | Phase 2 聚焦指定模块 |

## 输出

报告保存至：`{输出目录}/{YYYY-MM-DD HH:mm:ss}_{项目名}_概览.md`
