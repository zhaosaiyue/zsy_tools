# AI测试用例生成器

AI 驱动的测试用例生成器。深度解析需求/技术/核心用例文档，智能选择黑盒测试方法，生成 **XMind 兼容**的结构化测试用例树。

## 触发方式

关键词触发（在 Claude Code 对话中输入以下任意词即可激活）：

> 生成测试用例 / 写测试用例 / 帮我出用例 / 测试用例生成 / testcase / 出用例 / 用例 / 测试case / QA用例

## 工作流程

```
第一阶段：信息收集
  └─ 确认文档路径、测试范围、已有用例

第二阶段：测试分析
  └─ 判断需求类型（后端逻辑/配置/数据/前端/集成）
  └─ 确定覆盖维度、黑盒方法（等价类、边界值、判定表、场景法）

第三阶段：流式生成 & 写入
  └─ 按模块逐个生成，支持断点续写
```

## 输出格式

树形 Markdown，可直接导入 XMind：

```
# 功能模块名
## 场景分组
### 测试场景
#### 前置条件
#### 操作步骤
#### 预期结果
```

## 核心原则

- **反幻觉**：所有内容必须来源于文档，不凭空推断
- **原子性**：每条用例只验证一个行为
- **可执行**：步骤具体，不出现"正常输入"等模糊描述
- **可验证**：预期结果为系统具体反应（错误码、UI 文案、页面跳转）
