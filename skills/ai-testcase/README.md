# AI测试用例生成器

AI 驱动的测试用例生成器。深度解析需求/技术/核心用例文档，智能选择黑盒测试方法，生成结构化测试用例，**同时输出 `.md` 和 `.xmind` 两种格式**。

---

## 快速上手

**第一步：在对话中说出需求**

推荐直接 @ 相关文档：
```
@需求文档.pdf @技术文档.pdf 使用skill生成用例
```

或直接 @ 文档：

```
帮我生成测试用例
```

**第二步：按提示填写文档路径**

AI 会询问需求文档/技术文档路径、输出目录（留空自动存到文档同目录）、测试范围（留空 = 全部模块）。

**第三步：确认模块列表**

AI 解析完文档后输出识别到的模块，确认无误输入 `确认`，或直接输入 `直接出` 全自动跑完。

**完成后产出两个文件（同目录、同名）：**
- `2026-04-22 14:30:00_需求名.md` — 可直接阅读或手动导入 XMind
- `2026-04-22 14:30:00_需求名.xmind` — 双击用 XMind 打开

## 快捷指令

| 指令 | 效果 |
|------|------|
| `直接出` / `全自动` | 跳过所有确认节点，一气呵成 |
| `深度出` | 每模块生成前展示逻辑骨架供 review，自审追加幂等/反向/外部依赖失败场景 |
| `只要核心` | 只生成主流程用例，跳过字段级校验 |
| `重新解析` | 强制重新读取文档 |

## 工作流程

```
阶段 0：信息收集
  └─ 确认文档路径、输出目录、测试范围
  └─ 完整读取所有文档（PDF 支持分页读取）

阶段 1：测试分析
  └─ 判断需求类型（后端逻辑/配置/数据/前端/集成）
  └─ 按需生成判定表（条件维度 ≥ 3 时触发）

阶段 2：流式生成 & 写入
  └─ 逐模块生成，每模块完成立即写入 .md，支持断点续写
  └─ 识别到全链路场景时自动生成全链路场景节点（含正向 + 反向链路）
  └─ 全部完成后自动调用 md2xmind.js 生成 .xmind 文件
```

---

## 核心原则

- **反幻觉**：所有内容必须来源于文档，不凭空推断
- **原子性**：每条用例只验证一个行为
- **可执行**：步骤具体，不出现"正常输入"等模糊描述
- **可验证**：预期结果为系统具体反应（错误码、UI 文案、页面跳转）

## 初始化（首次使用）

`md2xmind.js` 依赖 `jszip`，首次使用前需在本目录安装依赖：

```bash
cd skills/ai-testcase && npm install
```
