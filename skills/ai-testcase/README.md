# AI测试用例生成器

AI 驱动的测试用例生成器。深度解析需求/技术/核心用例文档，智能选择黑盒测试方法，生成结构化测试用例，**同时输出 `.md` 和 `.xmind` 两种格式**。

## 触发方式

关键词触发（在 Claude Code 对话中输入以下任意词即可激活）：

> 生成测试用例 / 写测试用例 / 帮我出用例 / 测试用例生成 / testcase / 出用例 / 用例 / 测试case / QA用例

## 工作流程

```
阶段 0：信息收集
  └─ 确认文档路径、输出目录、测试范围
  └─ 完整读取所有文档

阶段 1：测试分析
  └─ 判断需求类型（后端逻辑/配置/数据/前端/集成）
  └─ 按需生成判定表（条件维度 ≥ 3 时触发）

阶段 2：流式生成 & 写入
  └─ 逐模块生成，每模块完成立即写入 .md，支持断点续写
  └─ 全部完成后自动调用 md2xmind.js 生成 .xmind 文件
```

## 输出

每次生成产出两个文件（同目录、同名）：

- `testcases_YYYY-MM-DD.md` — Markdown 格式，可手动导入 XMind 或直接阅读
- `testcases_YYYY-MM-DD.xmind` — 双击用 XMind 直接打开

## 核心原则

- **反幻觉**：所有内容必须来源于文档，不凭空推断
- **原子性**：每条用例只验证一个行为
- **可执行**：步骤具体，不出现"正常输入"等模糊描述
- **可验证**：预期结果为系统具体反应（错误码、UI 文案、页面跳转）

## 初始化（首次使用）

`md2xmind.js` 依赖 `jszip`，首次使用前需在本目录安装依赖：

```bash
cd skills/ai-testcase && npm install
```
